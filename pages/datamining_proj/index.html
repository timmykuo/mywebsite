Decision Tree¶

Decision tree includes implementation for nominal and continuous attributes, and uses both information gain and gain ratio as the split criterion (depending on a provided option below). Also implements stratified cross validation as described below. The code takes four options:

  1. Option 1 is the path to the data. If this is /a/b/someproblem then it loads /a/b/someproblem.names and /a/b/someproblem.data.
  2. Option 2 is to use cross validation or the full sample. If 0, use 10-fold cross validation. If 1, use the full sample.
  3. Option 3 is a nonnegative integer that sets the maximum depth of the tree (the number of tests on any path from root to leaf). If this value is zero, it grows the full tree. If this value is positive, it grows the tree to the given value. Note that if this value is too large, it will have the same effect as when the option is zero.
  4. Option 4 is a 0/1 option. If 0, it uses information gain as the split criterion. If 1, it uses gain ratio.

Naïve Bayes¶

Implementation of the nave Bayes algorithm discretizes continuous values by partitioning the range of the feature into k bins (value set through an option). Then, it replaces the feature with a discrete feature that takes value x if the original features value was in bin x. Uses m-estimates to smooth probability estimates. Uses logs whenever possible to avoid multiplying too many probabilities together. The main file is called nbayes. The code takes four options:

  1. Option 1 will be the path to the data (see Decision Tree Learner).
  2. Option 2 is to use cross validation or the full sample. If 0, use 10-fold cross validation. If 1, use the full sample.
  3. Option 3 (at least 2) is the number of bins for any continuous feature.
  4. Option 4 is the value of m for the m-estimate. If this value is negative, uses Laplace smoothing.

Note that m=0 is maximum likelihood estimation. The value of p in the m-estimate should be fixed to 1/v for a variable with v values.

T-test - Cross Validation¶

Now that we've taken a look at both of these algorithms, let's perform a t-test on the accuracies to see if one of them is superior with a 95% confidence. For this experiment, I'll use Laplace smoothing (m=1) for the naive bayes algorithm with 4 continuous bins and decision tree with max depth and using gain ratio. To run it yourself you can use the following commands:

python3 classifier.py nbayes bank 0 4 1
python3 classifier.py dtree bank 0 0 1

Note: For the experiment, I've modified the main method of classifier.py to print each fold. You can do this yourself in line 92/116 of classifier.py. Instead of printing the output at the end of the for-loop, indent it so that it is under the for loop and it prints each fold, aka "statistics.print_output(i)" instead of "statistics.print_output(USE_ALL_TABLES)".

Since we are using 10-fold cross validation, we will have 18 degrees of freedom 10+10-2. In addition, since are using 95% confidence, our t-value will be 2.10092204. Our null hypothesis is that there is no statistically significant difference between our two algorithms. Here are the results of the running the above commands:

In the above table, error rate was calculated through 1 - Accuracy. Now that we have our table of sample means and variances, let's perform the t-test:

$$t_{crit} = (m1 + m2) / \sqrt{(S1)^2/n1 + S2^2/n2}$$

where m is the mean, S is the standard deviation, and n is the number of samples for DTree and NBayes. After plugging in the numbers, we get

$$ t_{crit} = (0.50544 + 0.2282) / \sqrt{0.10621853299^2/10 + 0.00816214344193^2/10} = 0.27724 / 0.033688 = 8.2296 $$

Since $$ t_{crit} < t_{stat} $$ or $$ 2.10092204 < 8.2296 $$ we can reject our null hypothesis that there is no statistically significant difference between our two algorithms.

One possible reason for this is our dataset. As we explored earlier, the "bank" dataset has many continuous values and nominal attributes rather than binary attributes. As a result, the decision tree algorithm has a more difficult time splitting between these attributes since it uses a threshold value. On the other hand, our naive bayes algorithm creates multiple "bins" (depending on the input) and categorizes our examples according to the bins. This results in a much more accurate split between the examples, and thus overall accuracy of the resulting model. Although we could have used a lower depth of the tree for the decision tree algorithm, this would have been an inaccurate representation of the performance of the decision tree due to overfitting.

def nfold(n, file_base, seed, examples):
    #universal for examples using parser.py
    class_idx = -1
    folds = [[] for i in range(n)]
    positives = []
    negatives = []
    for example in examples:
        if example[class_idx]:
            positives.append(example)
        else:
            negatives.append(example)

    random.Random(seed).shuffle(positives)
    random.Random(seed).shuffle(negatives)

    fold_idx = 0
    while positives:
        folds[fold_idx].append(positives.pop())
        fold_idx += 1
        fold_idx %= n

    fold_idx = 0
    while negatives:
        folds[fold_idx].append(negatives.pop())
        fold_idx += 1
        fold_idx %= n

    return folds

def split_folds(folds):
    num_folds = len(folds)
    train_sets = [[] for i in range(num_folds)]
    test_sets = [[] for i in range(num_folds)]
    for omit in range(num_folds):
        for i in range(num_folds):
            if i == omit:
                test_sets[omit] += folds[i]
            else:
                train_sets[omit] += folds[i]

    return train_sets, test_sets

    #indexes in contingency table
    TRUE_POS = 0
    TRUE_NEG = 3
    FALSE_POS = 1
    FALSE_NEG = 2

    def __init__(self):
        self.cont_table = zeros(4)

    def add_to_table(self, tested_class, real_class):
        self.cont_table[self.cont_index(tested_class, real_class)] += 1

    def cont_index(self, tested_class, example_class):
        if example_class == tested_class:
            #example true, tested true
            if example_class:
                return self.TRUE_POS
            #example false, tested false
            else:
                return self.TRUE_NEG
        else:
            #example true, tested false
            if example_class:
                return self.FALSE_NEG
            #example false, tested true
            else:
                return self.FALSE_POS

    #fold: use cont_table resulting from fold number
    def accuracy(self):
        true_pos = self.cont_table[self.TRUE_POS]
        false_pos = self.cont_table[self.FALSE_POS]
        false_neg = self.cont_table[self.FALSE_NEG]
        true_neg = self.cont_table[self.TRUE_NEG]
        return (true_pos + true_neg) / (true_pos + true_neg + false_pos + false_neg)

    def precision(self):
        true_pos = self.cont_table[self.TRUE_POS]
        false_pos = self.cont_table[self.FALSE_POS]

"""

represent data stored in the C4.5 format.

An Example is a list of Features that complies to some
Schema, which describes feature types and values. An

Schema.

ExampleSets, Examples, and Schemas all implement the
necessary sequence methods so that syntax like:

>>> dataset[i][j]

gives the jth value of the ith example, and:

>>> for example in dataset: ...

iterates through examples in the dataset.

"""

class Feature(object):
    """
    Describes a feature by name, type, and values
    """

    class Type:
        """
        Enumerate types of features
        """
        CLASS = 'CLASS'
        ID = 'ID'
        BINARY = 'BINARY'
        NOMINAL = 'NOMINAL'
        CONTINUOUS = 'CONTINUOUS'

    def __init__(self, name, ftype, values=None):
        self.name = name
        self.type = ftype
        if (self.type == Feature.Type.ID or
                self.type == Feature.Type.NOMINAL):
            if values is None:
                raise Exception('No values for %s feature' % self.type)
            else:
                self.values = tuple(values)
        else:
            if values is None:
                self.values = None
            else:
                raise Exception('Values given for % feature' % self.type)
        self.tup = (self.name, self.type, self.values)

    def __cmp__(self, other):
        if self.tup > other.tup:
            return 1
        elif self.tup < other.tup:
            return -1
        else:
            return 0

    def __hash__(self):
        return self.tup.__hash__()

    def __repr__(self):
        return '<%s, %s, %s>' % self.tup

    def to_float(self, value):
        if value is None:
            return None
        if (self.type == Feature.Type.ID or
                self.type == Feature.Type.NOMINAL):
            return float(self.values.index(value))
        elif (self.type == Feature.Type.BINARY or
              self.type == Feature.Type.CLASS):
            if value:
                return 1.0
            else:
                return 0.0
        else:
            return value

class ExampleSet(collections.MutableSequence):
    """
    Represents a sequence of examples that
    all comply to the same schema
    """

    def __init__(self, schema_or_seq=None):
        self.examples = []
        if schema_or_seq is None:
            self.schema = None
        elif type(schema_or_seq) == Schema:
            self.schema = schema_or_seq
        else:
            self.schema = None
            try:
                self += schema_or_seq
            except:
                raise ValueError('Argument must be schema or list of examples')

    def to_float(self, mapper=None):
        """
        Convert this example set into a list of list
        of floats. Useful for constructing NumPy arrays.

        The mapper is a function applied to each example
        after it has been turned into a list of floats,
        and can be used to standardize data, for example.

        """
        return [ex.to_float(mapper) for ex in self]

    def schema_check(arg):
        """
        Wraps call to function f with a schema
        check on the argument specified by 'arg.'
        """
        def outer_wrapper(f):
            def inner_wrapper(self, *args, **kwargs):
                # Check Schema (if one exists)
                if (self.schema is not None and
                        args[arg].schema != self.schema):
                    raise ValueError('Schema mismatch')

                # Call function
                retval = f(self, *args, **kwargs)

                # If successful, set schema if none
                if self.schema is None:
                    self.schema = args[arg].schema

                return retval

            return inner_wrapper
        return outer_wrapper

    def __len__(self):
        return len(self.examples)

    def __iter__(self):
        return self.examples.__iter__()

    def __contains__(self, item):
        return self.examples.__contains__(item)

    def __getitem__(self, key):
        return self.examples[key]

    @schema_check(1)
    def __setitem__(self, key, example):
        self.examples[key] = example

    def __delitem__(self, key):
        del self.examples[key]

    @schema_check(1)
    def insert(self, key, example):
        return self.examples.insert(key, example)

    @schema_check(0)
    def append(self, example):
        super(ExampleSet, self).append(example)

    def __repr__(self):
        return '<%s, %s>' % (repr(self.schema), repr(self.examples))

    def __str__(self):
        return '[%s]' % ',\n '.join(map(str, self.examples))

def _parse_schema(schema_filename):
    """Parses C4.5 '.names' schema file"""
    features = []
    needs_id = True
    with open(schema_filename) as schema_file:
        for line in schema_file:
            feature = _parse_feature(line, needs_id)
            if feature is not None:
                if (needs_id and feature.type == Feature.Type.ID):
                    needs_id = False
                features.append(feature)

    # Fix the problem that the class feature is listed first in the
    # '.names' file, but is the last feature in the '.data' file
    try:
        features.remove(Feature.CLASS)
    except:
        raise Exception('File does not contain "Class" line')
    features.append(Feature.CLASS)

    return Schema(features)


def _parse_feature(line, needs_id):
    """
    Parse a feature from the given line. The second argument
    indicates whether we need an ID for our schema, in which
    case the first non-CLASS feature is selected.

    """
    line = _trim_line(line)
    if len(line) == 0:
        # Blank line
        return None
    if re.match(_BINARY_RE, line) is not None:
        # Class feature
        return Feature.CLASS
    colon = line.find(':')
    if colon < 0:
        raise Exception('No feature name found.')
    name = line[:colon].strip()
    remainder = line[colon + 1:]
    values = _parse_values(remainder)
    if needs_id:
        return Feature(name, Feature.Type.ID, values)
    elif len(values) == 1 and values[0].startswith('continuous'):
        return Feature(name, Feature.Type.CONTINUOUS)
    elif len(values) == 2 and '0' in values and '1' in values:
        return Feature(name, Feature.Type.BINARY)
    else:
        return Feature(name, Feature.Type.NOMINAL, values)


def _parse_values(value_string):
    """Parse comma-delimited values from a string"""
    values = list()
    for raw in value_string.split(','):
        raw = raw.strip()
        if len(raw) > 1 and raw[0] == '"' and raw[-1] == '"':
            raw = raw[1:-1].strip()
        values.append(raw)
    return values

    return line

def feature_value_threshold(examples, feature_idx):

    neg_weight = total_weight(negatives)
    min_entropy = entropy({0: left_bucket, 1: right_bucket}, pos_weight, neg_weight)

def C45(examples, nodes, remaining_depth, use_gain_ratio):
    total = total_weight(examples)
    num_positive = count_num_pos(examples)
    if num_positive == total:
        return LeafNode(True, 1.0)
    if num_positive == 0:
        return LeafNode(False, 0.0)
    # if there are no more attributes to divide on, return leaf node of majority class
    if not len(nodes) or not remaining_depth:
        return LeafNode(True if num_positive > total/2 else False, num_positive / total if num_positive > total/2 else 1 - (num_positive / total))

    best_node = nodes.pop()
    nodes.add(best_node)
    determining_factor = 0  # Best information gain or gain ratio
    positives, negatives = get_positives_negatives(examples)
    pos_weight = total_weight(positives)
    neg_weight = total_weight(negatives)
    current_entropy = entropy({0: [positives, negatives]}, pos_weight, neg_weight)
    #find the best node using gain_ratio/information_gain
    for node in nodes:
        buckets = node.test(examples)
        entropy_after = entropy(buckets, pos_weight, neg_weight)
        current_determining_factor = 0
        information_gain = current_entropy - entropy_after
        if use_gain_ratio:
            current_determining_factor = information_gain / node.entropy
        else:
            current_determining_factor = information_gain

        # store if max information gained
        if current_determining_factor > determining_factor:
            best_node = node
            determining_factor = current_determining_factor

    if determining_factor == 0:
        return LeafNode(True if num_positive > total/2 else False, num_positive / total if num_positive > total/2 else 1 - (num_positive / total))

    # split on attribute with most information gain and create children
    buckets = best_node.test(examples)
    children = {}
    for key in buckets:
        bucket = buckets[key]
        bucket_examples = bucket[0] + bucket[1]
        nodes.remove(best_node)
        children[key] = (C45(bucket_examples, nodes,
                            remaining_depth-1, use_gain_ratio))
        nodes.add(best_node)
    best_node.children = children
    return best_node

def attr_entropy(node, examples):
    buckets = node.test(examples)
    tot = len(examples)

    entropy = 0
    for key in buckets:
        bucket = buckets[key]
        bucket_length = len(bucket)
        bucket_prob = bucket_length / tot
        bucket_entropy = 0.0 if not bucket_prob else - \
            bucket_prob * math.log(bucket_prob, 2)
        entropy += bucket_entropy
    return entropy

# buckets is a dictionary that maps bucket_idx -> [positives,negatives]
def entropy(buckets, pos_weight, neg_weight):
    num_examples = 0
    for key in buckets:
        bucket = buckets[key]
        num_examples += pos_weight + neg_weight
    entropy = 0
    for key in buckets:
        bucket = buckets[key]
        num_pos = pos_weight
        num_neg = neg_weight
        bucket_size = num_pos + num_neg
        entropy += (bucket_size / num_examples) * bucket_entropy(num_pos, bucket_size)
    return entropy

# Calculate a bucket's entropy based on the number of positive examples and total examples
def bucket_entropy(num_pos, tot):
    if not tot:
        return 0.0
    pos_prob = num_pos / tot
    neg_prob = 1.0 - pos_prob
    pos_entropy = 0 if not pos_prob else pos_prob * math.log(pos_prob, 2)
    neg_entropy = 0 if not neg_prob else neg_prob * math.log(neg_prob, 2)
    return -(pos_entropy + neg_entropy)

def naive_bayes(examples, schema, num_bins, m_value):

        corr_dict = class_true if class_is_true else class_false

def get_continuous_ranges(examples, feature_idx, num_bins):
    ranges = []
    examples.sort(key=(lambda example: example[feature_idx]))
    bin_sizes = []
    leftover = len(examples) % num_bins
    for i in range(0, num_bins):
        bin_sizes.append(int(len(examples) / num_bins))
    for i in range(0, leftover):
        bin_sizes[i] += 1

    start = 0
    prev_range_end = n_infinity
    for idx in range(0, len(bin_sizes)):
        bin_size = bin_sizes[idx]
        range_start = prev_range_end
        range_end = infinity if idx == len(
            bin_sizes) - 1 else examples[start + bin_size - 1][feature_idx]
        prev_range_end = range_end
        ranges.append((range_start, range_end))
        start += bin_size - 1
    return ranges

def classify(example, schema, dict_class_true, dict_class_false, prob_true, prob_false):

            feature_name = schema[feature_idx].name

            # dict of the feature if class = true

                hypothesis_false += math.log(prob)
